连接（Connections）

你可以通过mongoose.connect()连接MongoDB。

mongoose.connect('mongodb://localhost/myapp');

至少你要连接到运行在默认端口（27017）的本地 myapp 数据库。 如果本地连接失败了，可以尝试把 localhost 替换为 127.0.0.1。 如果本地主机名称改变了，就可能会造成问题。

你可以在 uri 上加入一些更精确的参数：

mongoose.connect('mongodb://username:password@host:port/database?options...');

想要了解更多内容，请查看mongodb的uri连接配置。

操作缓存（Operation Buffering）

不用等待mongoose和MongoDB建立连接， Mongoose 就可以让你迅速地开始使用你的models。

mongoose.connect('mongodb://localhost/myapp');
var MyModel = mongoose.model('Test', new Schema({ name: String }));
// 有效
MyModel.findOne(function(error, result) { /* ... */ });

这是因为mongoose内部会缓存model函数的回调。 缓存使用非常方便，但也会造成不少困惑。 如果在没有连接数据库的时候使用model， Mongoose默认将 不会 抛出任何错误。

var MyModel = mongoose.model('Test', new Schema({ name: String }));
// 将会挂载，直到mongoose连接成功
MyModel.findOne(function(error, result) { /* ... */ });

setTimeout(function() {
  mongoose.connect('mongodb://localhost/myapp');
}, 60000);

为了禁用缓存，关闭schema上的bufferCommands选项。 如果在连接中已经有bufferCommands，或者你没有正确地打开一个连接，可以尝试关闭bufferCommands。 你也可以全局禁用bufferCommands：

mongoose.set('bufferCommands', false);

配置(Options)

connect方法接受一个传递到底层MongoDB驱动程序的options对象。

mongoose.connect(uri, options);

可以查看MongoDB Node.js的connect()驱动文档，来获取所有配置选项的列表。 Mongoose传递给驱动程序的过程中不会做任何修改，下面将解释三个例外。

  • bufferCommands - 这是一个mongoose特定的参数(不会传递给MongoDB驱动程序)，这个选项用于禁用mongoose缓存机制
  • user/pass - 用于验证的username和password。这些选项是mongoose特定的参数, 他们等同于MongoDB驱动器的 auth.user 和 auth.password 选项。
  • autoIndex - 默认情况，mongoose连接时，会在你的schema中自动构建索引， 这对于开发非常赞，但是因为构建索引会导致性能下降，所以对于大型产品的部署不是很理想，如果将 autoIndex 设置为false，对于任何关于这个连接的模型，mongoose将不会自动创建索引。

下面是一些对于mongoose调整非常重要的配置。

  • autoReconnect - The underlying MongoDB driver will automatically try to reconnect when it loses connection to MongoDB. Unless you are an extremely advanced user that wants to manage their own connection pool, do not set this option to false.
  • reconnectTries - If you're connected to a single server or mongos proxy (as opposed to a replica set), the MongoDB driver will try to reconnect every reconnectInterval milliseconds for reconnectTries times, and give up afterward. When the driver gives up, the mongoose connection emits a reconnectFailed event. This option does nothing for replica set connections.
  • reconnectInterval - See reconnectTries
  • promiseLibrary - sets the underlying driver's promise library
  • poolSize - The maximum number of sockets the MongoDB driver will keep open for this connection. By default, poolSize is 5. Keep in mind that, as of MongoDB 3.4, MongoDB only allows one operation per socket at a time, so you may want to increase this if you find you have a few slow queries that are blocking faster queries from proceeding.
  • bufferMaxEntries - The MongoDB driver also has its own buffering mechanism that kicks in when the driver is disconnected. Set this option to 0 and set bufferCommands to false on your schemas if you want your database operations to fail immediately when the driver is not connected, as opposed to waiting for reconnection.

Example:

const options = {
  useMongoClient: true,
  autoIndex: false, // 不会创建索引
  reconnectTries: Number.MAX_VALUE, // 从不终止重新连接
  reconnectInterval: 500, // 每500ms重连一次
  poolSize: 10, // 维持到10个socket连接
  // 如果没有连接，将立刻返回错误，而不是等待重连
  bufferMaxEntries: 0
};
mongoose.connect(uri, options);

回调(Callback)

connect()函数同样接收一个回调函数作为参数，并返回一个promise。

mongoose.connect(uri, options, function(error) {
  // 初次连接检查错误，回调函数无第二个参数
});

// 或者使用promise
mongoose.connect(uri, options).then(
  () => { /** ready to use. The `mongoose.connect()` promise resolves to undefined. */ },
  err => { /** handle initial connection error */ }
);

连接的字符串选项(Connection String Options)

你也可以通过连接的字符串，来配置驱动的参数， 就像URI中部分query字符串的参数一样。 这仅仅适用于传递给MongoDB驱动器的参数。 你不能在query字符串中，设置像bufferCommands这样的Mongoose特定参数。

mongoose.connect('mongodb://localhost:27017/test?connectTimeoutMS=1000&bufferCommands=false');
// 等同于：
mongoose.connect('mongodb://localhost:27017/test', {
  connectTimeoutMS: 1000
  // Note that mongoose will **not** pull `bufferCommands` from the query string
});

把配置放在query中在阅读的时候不是很友好。好处就是你只需要一个URI配置， 而不是例如socketTimeoutMS, connectTimeoutMS等等这样的单个选项。 最佳实践是将选项区分development和production， 就像在URI中字符串使用replicaSet 或者 ssl，在options对象中应该包含常量， 例如connectTimeoutMS 或 poolSize。

MongoDB文档拥有一个包含所有支持的字符串选项的列表。

关于KeepAlive的注意事项

对于长时间运行的应用，对于使用一个毫秒数来打开 keepAlive 经常是很谨慎的。 如果没有它，在一段时间后，你可能启动的时候会看到一个没有任何原因的 "connection closed" 的报错。 如果发生这种情况，请阅读以下这篇文章， 你可能就会开启keepAlive选项：

mongoose.connect(uri, { keepAlive: 120 });

副本集连接（Replica Set Connections）

如果要连接一个副本集，你需要传递一个以逗号（,）分隔的host列表， 而不是只连接一个host。

mongoose.connect('mongodb://[username:password@]host1[:port1][,host2[:port2],...[,hostN[:portN]]][/[database][?options]]' [, options]);

如果要连接单节点副本集，请指定 replicaSet 选项。

mongoose.connect('mongodb://host1:port1/?replicaSet=rsName');

多mongos支持（Multi-mongos support）

为了在分片集群中获取更高的可用性， 你也可以连接多个mongos实例， 在mongoose 5.x中，连接多个mongo， 不需要传递特定的配置选项。

// 连接2个mongo服务
mongoose.connect('mongodb://mongosA:27501,mongosB:27501', cb);

多连接（Multiple connections）

目前，我们已经知道如何使用Mongoose的默认连接去连接MongoDB。 有时候我们可能需要多对Mongo开放的连接，每一个都拥有不同的读写配置， 或者只是对于不同的数据库。 在这些情况下，我们可以利用mongoose.createConnection()函数， 其接收所有讨论过的参数， 并返回一个新连接。

const conn = mongoose.createConnection('mongodb://[username:password@]host1[:port1][,host2[:port2],...[,hostN[:portN]]][/[database][?options]]', options);

对象connection后面可以用于 创建和查找models。 Model对于单个连接总是拥有它的作用域。

当你调用 mongoose.connect() 时，Mongoose 将创建了一个 默认连接。 你可以通过mongoose.connection获取这个默认连接。

连接池（Connection pools）

无论是使用mongoose.connect 或者是 mongoose.createConnection创建的connection， 都会被一个内部的连接池储备起来，连接池最大连接数为5。 如果需要改变这个最大连接数，可以通过以下配置：

// 通过传入配置对象
mongoose.createConnection(uri, { poolSize: 4 });

const uri = 'mongodb://localhost/test?poolSize=4';
mongoose.createConnection(uri);

v5.x的配置变化（Option Changes in v5.x）

如果从4.x升级到5.x，你会看到以下警告，并且不能在4.x版本使用 useMongoClient 配置：

server/replset/mongos配置将不再支持，所有这些配置支持通过配置对象的第一级传入

在老版本的MongoDB驱动中，你必须为连接、副本集连接以及mongo连接，声明具体的配置：

mongoose.connect(myUri, {
  server: {
    socketOptions: {
      socketTimeoutMS: 0,
      keepAlive: true
    },
    reconnectTries: 30
  },
  replset: {
    socketOptions: {
      socketTimeoutMS: 0,
      keepAlive: true
    },
    reconnectTries: 30
  },
  mongos: {
    socketOptions: {
      socketTimeoutMS: 0,
      keepAlive: true
    },
    reconnectTries: 30
  }
});

在Mongoose v5.x 中，你可以在顶级声明这些配置， 而不用多余的嵌套。 这里是所有支持的配置列表。

// 和上面的代码是等效的
mongoose.connect(myUri, {
  socketTimeoutMS: 0,
  keepAlive: true,
  reconnectTries: 30
});
